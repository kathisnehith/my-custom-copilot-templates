# Implementation Planning Request

## Objective
Create a detailed, execution-ready implementation plan based on the attached materials and the goal described below.

## What do you want to build or implement?
${input:Describe the feature, system, or change you want to implement}

## Instructions
1. Review **all attached files** — requirements, research docs, existing code, and any related materials.
3. Analyze the current codebase architecture, patterns, dependencies, and conventions.
4. Produce a step-by-step implementation plan covering:
   - Architecture decisions and rationale
   - Module/component breakdown
   - Task decomposition with clear ordering and dependencies
   - File-level changes (new files, modified files)
   - API contracts, data models, and integration points
   - Testing strategy and acceptance criteria
   - Risks, constraints, and mitigation steps
5. Follow the implementation template from the skill at `/skills/implementation-template/skill.md`.
6. Present the summary plan for approval before saving the actual plan.
7. Save the approved plan to the location specified by the user with a descriptive file name.

## Planning Principles
- No code generation — planning and architecture only
- No assumptions — ask if information is missing
- Use existing research and codebase patterns
- Break work into practical, developer-actionable tasks

> **Attach your context files** (research docs, requirements, existing code, design docs, PDFs) to this prompt before running.
